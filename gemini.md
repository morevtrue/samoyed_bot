## Глобальные правила работы

### Окружение
+ Операционная система: **Windows**
+ Терминал: **PowerShell** (не cmd, не bash, не WSL)
+ Язык комментариев в коде: **русский**
+ Язык общения: **русский**

### Запрещено без явного запроса
+ Предлагать Docker, Kubernetes, WSL
+ Предлагать CI/CD, GitHub Actions
+ Менять версии пакетов и инструментов
+ Предлагать рефакторинг существующего кода
+ Добавлять новые зависимости
+ Менять архитектуру проекта

### Обязательно
+ Перед началом работы — изучить `/docs/` (приоритет у документации)
+ При неясности задачи — **СПРОСИТЬ**, не додумывать
+ Если есть несколько вариантов решения — **СПРОСИТЬ** о выборе
+ Указывать сложность изменений (1-10) при редактировании кода
+ Объяснять ЧТО изменено и ЗАЧЕМ

### Стиль работы
+ MVP-подход, не enterprise-архитектура
+ Простота важнее "правильности"
+ Работающий код важнее идеального кода
+ Один файл = одна ответственность
+ Избегать преждевременной оптимизации

### Формат предложений
Если предлагаешь изменение:
1. Опиши проблему
2. Предложи 2-3 варианта решения
3. Укажи плюсы/минусы каждого
4. **Дождись выбора** — не реализуй сразу
